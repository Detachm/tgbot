# Contract Assistant Bot KOL 合作介绍

## 一句话介绍

Contract Assistant Bot 是一个面向合约交易用户的 Telegram 交易助手 demo。它可以把 KOL 的策略变成可复刻、可推送、可模拟交易、可展示浮盈浮亏的 Telegram 体验，帮助用户从“看信号”进一步走到“跟随策略”和“自动交易”。

当前版本是演示环境，不会真实下单。所有成交、持仓和盈亏都是模拟数据，用于验证产品体验和合作模式。

## 为什么做这个产品

很多 KOL 的策略分发还停留在文字喊单、截图复盘、群内手动解释。用户看到信号后，还要自己理解方向、杠杆、仓位、止盈止损和下单入口，转化链路很长。

这个 bot 想解决三个问题：

- 信号标准化：把策略信号变成结构化卡片，而不是零散文字。
- 跟随低门槛：用户输入 6 位策略码即可复刻 KOL 策略。
- 反馈闭环：用户能看到模拟开仓、实时浮盈浮亏、平仓明细，而不只是收到一条信号。

## 当前 demo 已支持

- Telegram 内选择中文或英文。
- 默认加载热度最高策略并开始推送实时信号。
- 策略池展示，用户可以一键选择策略。
- KOL 策略码机制：KOL 发布 6 位策略码，用户输入后可复刻策略。
- 每 10 秒推送一张最新信号卡，旧信号卡会被删除，避免刷屏。
- 未绑定账户用户可以看到手动交易入口。
- 绑定 demo 账户后，用户可以选择模拟自动交易。
- 用户可以自定义杠杆、仓位、止盈、止损，保证金自动计算。
- 模拟成交后生成持仓浮盈浮亏卡片，每 5 秒刷新。
- 多个模拟持仓会分别展示浮盈浮亏。
- 用户可以模拟平仓，并收到完整平仓报告，包括价格、手续费、盈亏、余额、权益和可用余额。

## KOL 能怎么用

KOL 可以把自己的策略包装成一个策略码，例如 `A8K2QX`。用户在 bot 里输入这串 6 位码后，就能把该策略复制到自己的 bot 上下文里。

后续信号会按这个策略生成并推送，用户可以选择：

- 只接收信号提醒。
- 手动跳转交易页面。
- 使用 demo 账户模拟自动交易。
- 后续接入 API key 后开启真实自动交易。

这意味着 KOL 不只是发一条喊单，而是可以拥有一个轻量的策略分发和跟随入口。

## 当前是 demo，不是真实交易

为了安全和展示效率，当前版本不会真实调用交易所下单接口。

当前已完成的真实功能：

- Telegram bot 交互。
- 中英文切换。
- 菜单、按钮、策略池、策略码。
- 信号推送和旧信号清理。
- 用户状态、绑定状态、策略选择。
- 风控计算和订单意图。
- 模拟持仓、模拟盈亏、模拟平仓。

当前仍是 mock / demo 的部分：

- demo 账户余额。
- 模拟成交价格。
- 模拟订单号。
- 模拟持仓浮盈浮亏。
- 模拟平仓报告。
- LBank API key 绑定暂未接入真实交易。
- 不会真实下单、真实平仓或真实读取交易所余额。

## 后续可以为 KOL 定制什么

如果 KOL 认为这个方向值得深入，可以进一步定制：

- KOL 专属策略池和策略码。
- KOL 专属欢迎语和品牌展示。
- 策略参数模板，例如激进、稳健、防守模式。
- 策略表现页，例如胜率、最大回撤、历史信号、模拟收益。
- 用户分层，例如免费用户、VIP 用户、私域群用户。
- 跟单模式，例如手动确认、确认后交易、全自动交易。
- 风控规则，例如单笔仓位上限、最大杠杆、日亏损限制、暂停交易。
- 后续接入真实交易所 API key，实现真实自动交易。

## 真实交易的安全思路

真实交易阶段建议采用 API key 模式，而不是托管用户交易所账号密码。

原则如下：

- 用户只创建读取和交易权限的 API key。
- 不需要也不允许提现权限。
- 尽量开启 IP 白名单，只允许我们的交易服务器调用。
- 建议用户使用子账户或小额资金账户。
- API secret 加密保存，不在日志和消息中展示。
- 自动交易必须有风控上限和暂停按钮。
- 从 demo 到真实交易需要用户明确确认。

## 希望 KOL 重点反馈

我们希望 KOL 重点判断：

- 这个策略码复刻机制是否适合你的社群传播。
- 用户是否愿意先用 demo 跟随策略，再决定是否开启真实交易。
- 当前信号卡片展示的信息是否足够。
- 你希望策略里支持哪些参数。
- 你希望用户看到哪些策略表现数据。
- 是否有必要为你的社群做专属版本。

## 适合的下一步

建议先安排一次 20-30 分钟的 demo 演示。

演示内容包括：

- 用户首次进入 bot。
- 选择策略池。
- 输入 KOL 策略码复刻策略。
- 实时信号推送。
- 绑定 demo 账户。
- 模拟自动交易。
- 实时浮盈浮亏。
- 模拟平仓和交易明细。

如果 KOL 认可这个方向，下一步可以一起定义一版专属策略模板和合作试点范围。
